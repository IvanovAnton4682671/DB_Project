
для работы с Django нужно создать отдельную среду для проекта
для этого определяем каталог, в котором будем работать (переходим туда через cd)
создаём виртуальную среду: python -m venv .venv (среда с именем .venv)
запускаем среду .venv\Scripts\Activate.ps1
скачиваем Django в эту среду: python -m pip install Django
после окончания работы со средой её нужно деактивировать (deactivate)

создание проекта: django-admin startproject "название проекта"
для запуска проекта переходим: cd "название проекта", затем: python manage.py runserver
для остановки: ctrl+c

для создания своего приложения в проекте: python manage.py startapp "название приложения"
затем нужно его добавить в зависимости основного проекта, в settings, в INSTALLED_APPS
теперь можно написать какой-то ответ пользователю в новом приложении, в views.py
чтобы новое приложение что-то делало, нужно в основном проекте, в urls.py импортировать views из нового --->
---> проекта и в urlpatterns добавить соответствующий путь и связать с нужной функцией --->
---> (можно вставлять код html в ответ функций, чтобы грузилась страница, а не просто текст)

если в основном проекте, в settings, в TEMPLATES, в APP_DIRS стоит True, то мы можем создать --->
---> в нашем новом приложении папку templates, а в ней создавать страницы html, которые будут --->
---> передаваться в функции и показываться пользователю при правильной переадресации

для подключения бд к проекту нужно прописать команду: python manage.py migrate
а когда мы создаём свои модели для работы с бд - прописываем в консоли: python manage.py makemigrations
и выполняем полученный миграции: python manage.py migrate
